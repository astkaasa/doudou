# 批量管理安全评估

中后期长局会达到 80—120 架飞机和 40—80 条以上航线。批量操作的目标是减少重复点击，但不能用效率掩盖不可逆后果、非线性收益或合同影响。

## 本轮结论

本轮只实现“批量处置空闲飞机”。可选飞机必须同时满足：

- 不在交付中；
- 未分配给任何航线；
- 执行确认时仍存在且仍满足以上条件。

预览会列出自有机出售收入、租赁机季度租金节省、处置后现金、航线影响和处置后的租赁上限。执行采用整批重新校验；任一飞机状态变化时全批不执行，不做部分成功。

## 操作矩阵

| 操作 | 结论 | 原因 |
| --- | --- | --- |
| 处置空闲飞机 | 已实现 | 收入和节省可精确报价，且可保证 0 条航线受影响 |
| 批量停飞/复飞 | 延后 | 会影响机场合同、冷却期、客座率和飞机占用，不能只按当前筛选直接执行 |
| 批量关闭航线 | 不实现 | 不可逆，会释放飞机、终止合同并改变基地与网络收益 |
| 批量调价 | 延后 | 需求、竞争、枢纽和事件修正使收益变化非线性，需要逐线预测与差异审查 |
| 批量换机 | 延后 | 必须逐线校验航程、跑道、机场设施、减载和备降方案，且涉及替代机唯一分配 |
| 批量购机/租赁 | 沿用数量输入 | 现有市场已支持同机型 1—10 架报价，并明确交付、手续费、现金和租赁上限 |

## 后续立项条件

航线类批量操作只有在以下能力齐备后才应实现：逐线影响清单、机场合同结果、冷却和不可执行原因、静态收益差异、选择范围持久化、执行前二次快照校验，以及桌面与手机的撤销或恢复路径。

## 验收

领域测试覆盖混合出售/退租报价、重复选择、分配中、交付中、缺失飞机、过期选择和租赁比例警告。长局 Playwright 在桌面与手机上选择一架自有空闲机和一架租赁空闲机，验证影响预览、二次确认、整批执行、筛选刷新和结果横幅。
