# 整体产品与工程 Review

## 范围与结论

本轮覆盖游戏主流程、桌面与手机界面、季度反馈、机队和运营决策、领域计算、控制器动作、存档、测试、构建与单文件发布。Review 以玩家能否获得正确反馈、关键操作是否可达、规则口径是否一致为优先级，不把稳定模块的大规模重写视为优化。

当前没有阻断发布的已知问题。高优先级流程问题已修复并纳入自动化验证；四时代经济公式未在本轮改变，事件与新闻阶段记录的 1,600 局严格平衡矩阵仍是当前基线。

## 当前约束

### 离线单文件体积

生产 JavaScript 约 1.28 MB（gzip 约 270 KB），主要来源是约 1.57 MB 的 `src/data/airports.generated.js`。玩家入口必须是可直接双击的 `dist/standalone.html`，而当前构建器只内联入口模块；直接使用动态 `import()` 分块会破坏离线入口。因此 Vite 的 500 KB chunk 警告目前是已知约束，不是通过随意拆包即可修复的问题。

只有在 standalone 构建器能够递归内联模块图，或机场数据改为可验证的紧凑编码且仍保持稳定 ID、来源审计和离线能力后，才应立项处理该警告。

### 大模块

- `src/ui/map.js` 仍约 1,040 行，但包含投影、循环世界、触控命中、缩放补偿和缓存。下次新增地图行为时，优先按“投影与几何 / 交互状态 / DOM 渲染”拆分，并保持当前性能和双视口测试。
- `src/domain/subsidiaries.js` 同时承载投资规则、季度结算和公司估值。只有修改其中一条规则时再沿现有领域边界逐步拆分，避免一次性重写存档相关逻辑。
- `src/domain/invariants.js` 和 `src/domain/save.js` 较长是因为静态数据校验与多版本迁移。后续新增存档版本时，应把新版本迁移写成独立步骤，并继续用旧存档样本覆盖，而不是改写既有迁移链。
- `src/styles/app.css` 构建后约 111 KB。源码内静态内联样式只保留地图坐标这一类运行时几何值；按组件拆 CSS 只有在能改善所有权和审查成本时再做，不能只为了文件行数移动代码。

## 明确延后

- 机场时刻交易继续按 `docs/airport-slot-feasibility.md` 延后，需先具备 AI 竞价、价格发现、存档迁移和双视口交互方案。
- 不为消除文件行数或构建警告引入 React、Vue、TypeScript、状态管理库或第二套构建系统。
- 不在没有新玩法需求的情况下重做地图、存档或子公司系统。

## 后续路线图

| 优先级 | 触发条件 | 工作与验收 |
| --- | --- | --- |
| P1 地图触控 | 下次修改缩放、循环卷轴、机场点或手势 | 按投影几何、交互状态和 DOM 渲染逐步分离；增加拖动、双指缩放、循环衔接测试，并做窄屏真机误触检查 |
| P2 生成数据体积 | gzip 主包比当前基线增长 10% | 在生成器中对比紧凑数组、字段字典和年代索引；只有 gzip 至少下降 15%，且数据审计与 standalone 均通过时采用 |
| P3 领域模块 | 子公司、存档或校验模块出现实质规则变更 | 一次只拆一个行为边界，先补测试再移动代码；外部 API、迁移顺序和存档结果保持不变 |
| P4 长期玩法 | 修改经济常量、主线目标、事件收益或机场容量 | 重跑严格 1,600 局矩阵，并增加 200 季以上沙箱样本，检查现金膨胀、拥堵、退役和 AI 航网规模 |
| P5 人工体验 | 发布候选版本 | 对桌面和手机执行视觉矩阵；地图手势、键盘全流程、屏幕阅读器名称与低对比文本保留人工抽查 |

## 发布验收

本轮交付必须同时满足：

```bash
npm run check
npm run data:audit
npm run test:e2e
node scripts/verify-standalone-browser.mjs
git diff --check
```

本轮结果：

- `npm run check`：44 个测试文件、248 项测试通过；生产构建、standalone 生成和发布清单校验通过。
- `npm run data:audit`：123 座城市、1,015 座机场通过审计，0 条警告。
- `npm run test:e2e`：桌面与手机共 8 项浏览器测试通过。
- `node scripts/verify-standalone-browser.mjs`：`file://` 单文件入口冒烟通过。
- 人工视觉抽查：1280×720 与 390×844 的主界面、航线创建、运营管理和季度财报无横向溢出、遮挡或不可达操作；浏览器控制台无错误。
- 经济与 AI 公式未改变；沿用事件与新闻阶段已通过的 1,600 局、80/80 组合严格平衡矩阵。

经济公式、目标或 AI 策略发生变化时，额外执行：

```bash
npm run balance:acceptance -- --workers 8 --strict
```

双视口场景和人工失败标准见 `docs/visual-regression.md`。只有上述检查通过、玩家可见变更进入 `CHANGELOG.md`、未提交文件边界明确后，本轮 Review 才算完成。
