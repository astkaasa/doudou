# 事件与新闻系统

## 内容通道

当前报纸内容分为四条通道，各自承担不同职责：

| 通道 | 数据来源 | 触发方式 | 主要作用 |
| --- | --- | --- | --- |
| 随机新闻 | `src/data/news.js` | 年代、季度、玩家状态过滤后随机抽取 | 制造经营波动与时代气氛 |
| 固定日期新闻 | `src/data/news.js` 的 `scheduled` 条目 | 指定年份和季度必定出现 | 呈现机场更名等确定事项 |
| 历史盛事 | `src/data/megaEvents.js`、`src/data/worldCups.js` | 在盛事前后固定窗口出现 | 为主办城市和区域网络提供可预期需求 |
| 机型目录 | `src/domain/aircraftNews.js` | 每年 Q1 根据采购窗口生成 | 解释可购机型变化和机场适配要求 |

机场启停与迁场继续由 `src/domain/airportRelocations.js` 管理。需要玩家作出选择的事项不能降级为普通报纸新闻。

## 随机新闻规则

新闻可使用以下元数据：

- `startYear` / `endYear`：连续年代窗口。
- `years` / `quarters`：精确年份或季度集合。
- `scheduled`：符合时间条件时必定出现，不参与随机抽取。
- `requiresRoutes`：只有玩家已有航线时才可入池。
- `requiresAirportRoutes`：只有玩家已有具体机场端点航线时才可入池。

随机新闻不会连续两个季度抽到同一来源标题。机场类 `effectFn` 可以返回新的 `title`、`desc` 或 `effect`，用实际机场代码替换通用文案，但所有经营影响仍必须通过 `state.activeModifiers` 生效。

## 历史盛事规则

盛事的 `cityId` 必须引用稳定城市 ID。场馆城市不在城市表中时，可以映射到承担主要国际客流的可玩航空门户，但正文必须写明实际场馆城市。例如：

- 1994 年世界杯决赛在帕萨迪纳举行，玩法门户使用洛杉矶。
- 2002 年世界杯决赛在横滨举行，玩法门户使用东京。

盛事按峰值季度记录：奥运和世博通常使用开幕季度，世界杯使用决赛季度。需求依次影响主办门户城市、同区域城市和远端枢纽，不再使用全球统一需求加成。

世界杯采用短于奥运和世博的赛事型曲线，并只影响主办门户及较弱的同区域溢出；远端航线不会无条件获得需求红利。多年行业趋势、文化关注和技术普及类随机新闻可以只影响行业股票，不应反复叠加为玩家航线的稳定利润来源。

## 机型新闻规则

`serviceStart` / `serviceEnd` 是包含首尾年份的游戏采购窗口，不等同于现实中的首飞、商业首航、停产或全行业退役年份。机型在 `serviceEnd` 当年仍可购买，退出采购目录的新闻在下一年 Q1 发布。机型新闻因此只使用以下措辞：

- 加入或退出采购目录。
- 开放或停止接受新订单。
- 现有机队继续运营和转售。

架空机型必须保留 `fictional: true`。采购新闻需要展示座位、航程、最低跑道和基础设施等级，使新闻同时承担经营信息入口。

## 扩充检查

新增内容前至少确认：

1. 该内容属于随机、固定日期、盛事还是玩家决策事项。
2. 年份、季度、城市和机场引用有可靠来源。
3. 早期剧本不会出现现代术语或技术。
4. 无相关航线时不会生成无法生效的经营承诺。
5. 数值影响使用修正器，并覆盖边界测试和批量平衡模拟。
